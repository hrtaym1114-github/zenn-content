# ベストプラクティス

## 推奨事項

1. **推論レベルを適切に選択する**: 簡単なタスクに`xhigh`を使うのはコストの無駄。タスクの難易度に応じて`low`〜`high`を使い分ける
2. **Tool Searchを活用する**: 5個以上のツールを使う場合はTool Searchを有効化してトークン消費を削減
3. **Batch + Flexで半額**: リアルタイム性が不要なタスクはバッチ処理で半額に
4. **長文コンテキストは必要な時だけ**: 272Kトークンを超えると2倍料金。本当に必要な場合のみ使用
5. **マルチエージェントを活用**: Codexの並列実行で開発速度を向上
6. **Playwright CLIを優先**: MCP版（114Kトークン）よりCLI版（27Kトークン）が75%効率的

## アンチパターン

:::message alert
以下はコストや効率の面で避けるべきパターンです。
:::

- **xhighモードをリアルタイム応答に使わない**: 8分以上かかる可能性があり、UXが悪化
- **全ツール定義を毎回送らない**: Tool Searchで必要なものだけ検索する
- **1Mコンテキストを常用しない**: コスト増大の原因。必要な部分だけを送る
- **GPT-5.4 ProをGPT-5.4で十分なタスクに使わない**: 12倍のコスト差

## コスト最適化のヒント

| 推論レベル | 適用場面 |
|-----------|---------|
| `none` / `low` | 単純な変換・フォーマット・翻訳 |
| `medium` | 標準的なコーディング・文章生成 |
| `high` | 複雑なバグ修正・設計判断 |
| `xhigh` | 研究レベルの問題のみ |

大量のタスクは半額の**Batch + Flex**で処理しましょう。
