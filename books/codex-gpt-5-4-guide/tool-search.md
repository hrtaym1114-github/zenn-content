# Tool Search（ツール検索）

## 概要

Tool Searchは、GPT-5.4で導入されたAPI向けの新機能で、多数のツール定義を効率的に扱うための仕組みです。

## 従来の問題

従来のアプローチでは、利用可能なすべてのツール定義をプロンプトに含める必要がありました。MCP（Model Context Protocol）で多数のサーバーを接続すると、ツール定義だけでコンテキストウィンドウの大部分を消費してしまう問題がありました。

## 動作原理

Tool Searchでは以下のように動作します。

1. モデルに**軽量なツール一覧**と**ツール検索機能**を提供
2. モデルが必要に応じてツール定義を**オンデマンドで検索**
3. 必要なツールの定義だけを会話に追加

```
従来: 全ツール定義をコンテキストに含める → トークン大量消費
Tool Search: 必要時にだけ検索 → 47%トークン削減
```

## ベンチマーク結果

ScaleのMCP Atlas Benchmarkで測定した結果は以下の通りです。

- **36個のMCPサーバー**を有効にした250タスク
- Tool Search構成は全ツール定義を直接公開する構成と**同じ精度**を維持
- **トークン使用量47%削減**

## 実用的な意味

- **コスト削減**: トークン消費の大幅な削減
- **スケーラビリティ**: 数十〜数百のツールを問題なく扱える
- **MCP連携の強化**: 多数のMCPサーバーを同時に利用可能に
